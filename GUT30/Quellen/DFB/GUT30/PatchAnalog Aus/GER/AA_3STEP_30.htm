AA_3STEP_30

Allgemein

Der abgeleitete Funktionsbaustein AA_3STEP_30 wird genutzt, um die Stellgröße als Größer und Kleiner Bit auszugeben.

Intern nutzt der Baustein einen 3 Punkt Regler und benötigt eine Rückmeldung der Stellung über einen AE Baustein.

Der DFB ist wie folgt implementiert:

Software

Implementiert als

Unity Pro

DFB AA_3STEP_30
Funktionsbeschreibung

Der Baustein Skaliert einen Ausgangswert in einen Int Wert.

Verhalten der Ausgänge

Verhalten der Ausgänge OUT_POS und OUT_NEG:

td        Zeitdauer


Wenn die Abweichung (DEV = AE.X - Y) über dev_hl hinaus ansteigt, wird der logische Ausgang OUT_POS auf 1 gesetzt. Wenn die Abweichung geringer wird, wird OUT_POS jedoch nur dann auf Null gesetzt, wenn die Abweichung dev_hl - hyst unterschreitet.

Wenn die Abweichung dev_ll unterschreitet, wird der logische Ausgang OUT_NEG auf 1 gesetzt. Wenn die Abweichung erneut ansteigt, wird OUT_NEG nur dann auf Null gesetzt, wenn die Abweichung über dev_ll + hyst hinaus ansteigt.

HINWEIS: Um eine fehlerfreie Funktionsweise des Bausteins zu gewährleisten, dürfen die Ausgänge OUT_NEG und OUT_POS nicht invertiert werden.

Folgende Verhalten werden angezeigt:

  • Y liegt außerhalb des Bereichs [pv_inf, pv_sup]: Y wird in diesem Fall auf pv_inf bzw. pv_sup begrenzt.
  • dev_ll > 0 oder dev_hl < 0: der Baustein verwendet den Wert 0
  • hyst liegt außerhalb des Bereichs [0, Minimum (dev_hl, -dev_ll)]: der Baustein verwendet einen auf Null bzw. auf Minimum (dev_hl, -dev_ll) begrenzten Wert
